Created2024-02-09|Updated2024-02-09|程式

有鑑於我DP實在太爛，所以我為了提升自己的能力，所以開始了這次挑戰，規則如下:

  1. 每天刷三題分類為DP的題目
  2. 如果寫九十分鐘內寫不出來可以看講詳解
  3. 刷題過程中可以查找語法

Day 1

322.Coin Change(Medium)
題目給定一串數組，代表硬幣的幣值，並且給出一個數字，要求我們使用最少此數組中的硬幣來組成這個數值。
這一題沒有限制使用硬幣的上限，並且沒有拆分硬幣，所以不是greedy，然後大問題可以拆成子問題，dp[5] 可以拆成 dp[1] + dp[4] 和 dp[2] + dp[3]，以此類推，所以是DP。

解題思路:

創建一個長度為amount+1的陣列dp，dp[i]代表數值i的最佳解，然後去遍歷0~amount。

轉移式: dp[i] = min(dp[i], dp[i - coins[j]] + 1)

62. Unique Paths(Medium)
這一題是標準的DP填表問題，題目給定一個座標(m, n)，我們的初始點為(0, 0)，我們只能往下或往右走，然後我們要找出所有能到達目的地的路線數。

解題思路:

宣告一個m * n 的陣列 dp，然後把 i = 0 和 j = 0的路線格都先給定為 1(因為只能往下跟往右走，所以這幾個座標都只有一條路徑能走)，之後就可以填表。

轉移式: dp[i][j] = dp[i-1][j] + dp[i][j-1]

5. Longest Palindromic Substring(Medium)
題目要求我們找出最長的迴文子字串，這一題是經典的填表解。

解題思路:

宣告一個 s.size() * s.size() 的陣列dp，dp[i][j] 代表 i~j 是否為一個迴文，然後有三種情況

  • i == j：這種情況代表只有一個字元，長度為1的迴文
  • s[i] == s[j] && i = j + 1： 這種情況代表只有兩個相同字元，長度為2的迴文
  • s[i] == s[j] && dp[i+1][j-1]： 這種情況代表這兩個字元中間的字串也是迴文，所以加上兩個相同字元也是迴文

Day 2

1143. Longest Common Subsequence(Medium)

題目給定兩個字串，字串長度分別為 m 和 n，要我們找出他們共同且最長的subsequence。

解題思路:

宣告一個 m * n 的陣列dp，dp[i][j] 代表text1前i個字元組成的字串與text2前j個字元所組成的字串之間所擁有共同且最長的subsequence。

轉移式:

if (text1[i] == text2[j]) dp[i][j] = dp[i-1][j-1] + 1;

if (text1[i]  !=  text2[j]) dp[i][j] = max(dp[i-1][j],  dp[i][j-1]);

221. Maximal Square(Medium)
題目給定一個m*n的matrix，裡面只由 ‘0’ 和 ‘1’ 組成，我們要找出全部都由 ‘1’組成的最大正方型區域。

解題思路:

宣告一個 m * n 的陣列dp，dp[i][j]代表由matrix[i][j]為右下角組成的最大正方形邊長。

轉移式:

  • 當 matrix[i][j] == 0時，那就不可能以它當作正方形右下角，所以dp[i][j] = 0。
  • 當 matrix[i][j] == 1時，dp[i][j] = min(dp[i-1][j-1], dp[i-1][j], dp[i][j-1]) + 1

576. Out of Boundary Paths(Medium)

題目給定一個 m * n的地圖和足球的起始座標，然後給定一個最大移動次數，我們每次可以將球往上下左右移動一格，我們要找出在最大移動次數內，我們能將球踢出場外的方法數。

解題思路:

宣告一個m*n的陣列dp，dp[i][j]代表有幾種方式可以移動到(i, j)這一點，然後使用巢狀迴圈遍歷整個陣列，當點在邊界時，方法數 += dp[i][j]，為了加速，所以我多開了一個vector去存取每一步的資訊。

97. Interleaving String(Medium)
題目給定三個字串 s1, s2, s3，問我們能不能將s3由s1和s2的substring組成。

解題思路:

我們可以觀察s3，如果s3的最後一個字元一定是使用s1或s2的最後一個字元，所以我們可以找出子問題為當前s3前綴的最後一個字元是否跟s1和s2的最後一個字元相同，如果相同則要去看前面是否能組成s3的前綴，我宣告陣列dp，dp[i][j]代表s1前i個字元和s2前j個字元是否能夠組成s3前i+j個字元，判斷式只需看dp[i-1][j] 或 dp[i][j-1]是否為True即可。

115. Distinct Subsequences(hard)
題目給兩個字串s、t，要我們找出使用s的subsequence組出t的方法數。

解題思路:

宣告一個陣列dp，dp[i][j]代表t的前i個字元能被s的前j個字元組出來的方法數，而在解subsequence問題時，我們通常只需在意最後一個字元所以我們可以推出轉移式。

轉移式:

if(s[j] == t[i]) dp[i][j] = dp[i][j-1] + dp[i-1][j-1]
else dp[i][j] = dp[i][j-1]

72. Edit Distance(Medium)
題目給兩個字串 word1、word2，我們可以對word1進行insert、delete、replace，我們要找出使用最少次操作數。

解題思路:

516. Longest Palindromic Subsequence(Medium)
題目給定一個字串，要求我們找出最長palindromic subsequence的長度。

解題思路:

這一題是標準的迴文dp裸題，宣告一個陣列dp，dp[i][j]代表s[i] ~ s[j]這個子字串中最長palindromic subsequence的長度，跟常見的迴文問題相同，只看子字串的開頭結尾，當s[i] ==s[j]時，dp[i][j] = dp[i+1][j-1] + 2，如果不同，dp[i][j] = max(dp[i+1][j-1], dp[i+1][j], dp[i][j-1])，但有一個例外，當i == j時，迴文長度為1。

解法:

這解法有一個地方需要注意，因為我們的轉移式會用到 dp[j+1][i-1]、dp[j+1][i]、dp[j][i-1]，所以我們遍歷的順序必須是 row : 由下到上，column:由左到右，巢狀迴圈中第二層迴圈為row的index。

1092. Shortest Common Supersequence(hard)
題目給定兩個字串str1和str2，要我們找出一個最短字串s，且str1和str2都是這個s的subsequence。

解題思路:

答案的長度其實是 str1.size() + str2.size() - LCS(str1, str2)，所以我們要先建立LCS的表，然後利用LCS的表去回溯字串，回溯寫法跟平常不太一樣，要將回溯路徑上的字元全部都記錄下來，然後再加上沒有使用的的所有字元就是答案。

Day 5

96. Unique Binary Search Trees(Medium)
這題給定一個數字n，要我們找出數列[1, 2, 3, …, n]能夠組成幾個BST。

解題思路:

宣告一個長度為n+1的陣列dp，dp[i]代表i可以被組成多少個BST。

算法:

1~i輪流當root，每個root的可能性會是左子樹數值的數量 * 右子樹數值的數量。

e.g. 5當root，n為10，5的左子樹總共有4個也就是dp[4]，右子樹有5個為dp[5]，所以5當root時，能組成BST的數量為dp[4] * dp[5]。

1043. Partition Array for Maximum Sum(Medium)
題目給定一個陣列arr和整數k，要求我們分割arr，每次分割長度最多為k，分割完後，subarray的中的所有值會變成此subarray中最大的數字，我們要找出分割後，arr最大的總和。

解題思路:

這一題明顯是使用DP解，宣告一個長度為n+1的陣列dp，dp[i]代表前i個數字分割後所能獲得之最大值。以dp[i]來說，我們從最後一個數字往前分割j個數字，那我們就會被分成兩段，後半段可以找出其最大值，然後再加上前半段的最大值，就能找出答案，而前半段的最大值可以直接拿dp[i-j]的值。
